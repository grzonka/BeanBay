    1. **Header:** BeanBay name, one-line description ("Coffee optimization powered by Bayesian learning"), beanbay.coffee link
    
    2. **What is BeanBay?** — 2-3 paragraphs explaining:
       - Phone-first web app for optimizing coffee recipes
       - Uses BayBE (Bayesian optimization) to learn from each brew
       - Self-hosted, runs on any Docker host (Unraid, Synology, Raspberry Pi, etc.)
       - Currently optimized for espresso, with filter/immersion brewing coming in v2
    
    3. **Features** — bullet list:
       - Bean management (create, select, per-bean history)
       - BayBE-powered recipe recommendations with transparent reasoning

       - Optimization progress visualization
       - Parameter exploration heatmaps
       - Cross-bean comparison
       - Brew statistics dashboard
       - Mobile-first dark UI (designed for use at the espresso machine)
    
    4. **Quick Start** — Docker deployment:
       ```bash
       docker run -d --name beanbay -p 8000:8000 -v beanbay-data:/data ghcr.io/grzonka/beanbay:latest
       ```
       Then visit `http://your-server:8000`
    
    5. **Docker Compose** — reference docker-compose.yml
    
    6. **Unraid** — link to Unraid template (coming in Phase 9), or "install via Community Apps"
    
    7. **Development Setup:**
       ```bash
       git clone https://github.com/grzonka/beanbay.git
       cd beanbay
       python -m venv .venv
       source .venv/bin/activate
       pip install -e ".[dev]"
       alembic upgrade head
       uvicorn app.main:app --reload
       ```
    
    8. **Running Tests:**
       ```bash
       pytest tests/ -v
       ```
    
    9. **Tech Stack** — FastAPI, Jinja2/htmx, SQLite, Chart.js, BayBE, Docker
