# Plan 03-01: Brew Router & Optimization Loop

**Phase:** 3 — Optimization Loop
**Goal:** Implement the complete espresso optimization cycle: get recommendation, brew, rate, recall best.

## Requirements Covered

- OPT-01: Request BayBE-powered recommendation for active bean
- OPT-02: See recommended params in large scannable text
- OPT-03: See brew ratio (dose:yield) alongside recommendation
- OPT-04: Submit taste score (1-10, 0.5 steps)
- OPT-05: Mark shot as failed (choked/gusher), auto-set taste to 1
- OPT-06: View and re-brew current best recipe with one tap
- SHOT-03: Record actual extraction time in seconds

## Tasks

### 1. Brew Router (`app/routers/brew.py`)

Endpoints:
- `GET /brew` — Main brew page. Shows "Get Recommendation" button if no pending recommendation. If active bean has measurements, also shows "Repeat Best" button.
- `POST /brew/recommend` — Triggers BayBE recommendation for active bean. Stores recommendation in session/template, redirects to display.
- `GET /brew/recommend/{recommendation_id}` — Display the recommendation with large params + rate form.
- `POST /brew/record` — Records measurement (taste, extraction_time, is_failed, recipe params as hidden fields). Saves to both SQLite and BayBE campaign. Redirects back to brew page.
- `GET /brew/best` — Shows the best recipe (highest taste score) for active bean.

All endpoints require active bean (redirect to /beans if none).

### 2. Brew Templates

- `brew/index.html` — Main brew page with action buttons
- `brew/recommend.html` — Shows 6 recipe params in large text + ratio + rate form
- `brew/best.html` — Shows best recipe with "Brew Again" button
- `brew/_recipe_card.html` — Reusable partial: 6 params in large scannable format + ratio

### 3. Brew-specific CSS

- `.recipe-param` — Large, scannable parameter display (readable from arm's length)
- `.recipe-value` — Big numbers for recipe values
- `.score-slider` — Taste score input (1-10, 0.5 steps)
- `.failed-toggle` — Prominent failed shot toggle
- `.ratio` — Brew ratio display

### 4. Wire into app

- Add `from app.routers import brew` and `app.include_router(brew.router)` in `app/main.py`
- Add "Brew" nav link to `base.html`
