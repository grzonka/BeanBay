<task type="auto">
  <name>Task 2: Add desktop sidebar CSS with ≥768px media query</name>
  <files>app/static/css/main.css</files>
  <action>
  Add a comprehensive `@media (min-width: 768px)` block at the END of `main.css` (after all existing styles, before EOF). This media query transforms the mobile layout into a desktop sidebar layout.

  **Desktop layout rules inside `@media (min-width: 768px)`:**

  1. **Hide mobile top bar:**
     ```css
     .nav { display: none; }
     ```

  2. **App layout — sidebar + content flex:**
     ```css
     .app-layout {
       display: flex;
       min-height: 100dvh;
     }
     ```

  3. **Convert drawer to permanent sidebar:**
     ```css
     .nav-drawer {
       position: static;
       transform: none;
       width: 240px;
       min-width: 240px;
       height: 100dvh;
       position: sticky;
       top: 0;
       border-right: 1px solid var(--border);
       background: var(--bg-secondary);
       padding-top: var(--spacing-lg);
       z-index: auto;
       overflow-y: auto;
     }
     ```
     The `position: static` + override to `sticky` keeps the sidebar in flow but sticky as you scroll.

  4. **Hide overlay on desktop:**
     ```css
     .nav-overlay { display: none !important; }
     ```

  5. **Show brand in sidebar:**
     ```css
     .nav-drawer-brand {
       display: block;
       padding: var(--spacing-sm) var(--spacing-lg) var(--spacing-lg);
       font-size: var(--text-lg);
       font-weight: 700;
       color: var(--accent);
       text-decoration: none;
     }
     ```

  6. **Main content fills remaining width:**
     ```css
     .main {
       flex: 1;
       min-width: 0;
       padding: var(--spacing-lg) 0 var(--spacing-xl);
     }
     ```

  7. **Container max-width uncapped for desktop:**
     ```css
     .container {
       max-width: 960px;
     }
     ```
     This replaces the 480/540px cap with 960px on desktop, giving content room to breathe while still not spanning ultra-wide monitors.

  8. **Active bean in sidebar — proper truncation (NAV-03):**
     ```css
     .nav-drawer-bean {
       padding: var(--spacing-sm) var(--spacing-lg);
     }
     .nav-drawer-bean a {
       overflow: hidden;
       text-overflow: ellipsis;
       white-space: nowrap;
       display: block;
       max-width: 160px;
     }
     ```

  **ALSO add outside the media query (mobile default):**
  ```css
  .nav-drawer-brand {
    display: none;
  }
  ```
  This hides the drawer brand on mobile (where the top bar brand is visible instead).

  **Update the existing `@media (min-width: 480px)` block:**
  - Keep it but it's now only relevant for the 480-767px range. No changes needed — the 768px query will override `.container` max-width.

  **Verify no conflicting styles:**
  - The drawer's mobile styles (position: fixed, transform: translateX(-100%), etc.) are overridden by the desktop query's `position: static; transform: none;`
  - The `.nav-drawer.open` class still works on mobile but is irrelevant on desktop since the drawer is always visible
  </action>
