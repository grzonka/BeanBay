  Helpers:
  - `_get_active_bean(request, db)` — same pattern as other routers (read from cookie)
  - `_compute_convergence(measurements_list)` — takes a list of measurement dicts (ordered by created_at asc), returns:
    ```python
    {
        "status": str,     # "getting_started" | "early_exploration" | "narrowing_in" | "refining" | "near_optimal"
        "label": str,      # Human-readable label
        "description": str, # Explanation text
        "color": str,      # CSS class suffix for badge
    }
    ```
    Rules:
    - n < 3: status="getting_started", label="Getting started", description="Pull a few more shots to give the optimizer enough data to start learning.", color="muted"
    - n < 8: status="early_exploration", label="Early exploration", description="The optimizer is mapping out the parameter space. Each shot teaches it something new.", color="blue"
    - n >= 8, check last 3 non-failed shots: if max taste in last 3 > best taste before last 3:
      status="narrowing_in", label="Narrowing in", description="Recent shots are hitting new highs. The optimizer is finding promising territory.", color="amber"
    - n >= 8, no improvement in last 5+: status="near_optimal", label="Likely near optimal", description="The optimizer hasn't found anything better recently. Your current best recipe may be close to the limit for this bean.", color="green"
    - n >= 8, no improvement in last 3-5: status="refining", label="Refining", description="The optimizer is fine-tuning. It may still find small improvements.", color="amber"

  - `_build_chart_data(measurements_list)` — takes measurement dicts ordered by created_at asc, returns dict suitable for Chart.js:
    ```python
    {
        "labels": [1, 2, 3, ...],  # Shot numbers
        "individual_scores": [6.5, 7.0, 5.0, ...],  # Each shot's taste
        "cumulative_best": [6.5, 7.0, 7.0, ...],  # Running max
        "failed_indices": [2, 5, ...],  # Indices of failed shots (0-based)
    }
    ```
    Only include non-failed shots in cumulative_best calculation, but show all shots (failed shots are visually distinct).

  Main route:
  ```python
  @router.get("", response_class=HTMLResponse)
  async def insights_page(request: Request, db: Session = Depends(get_db)):
      """Insights page — optimization progress and convergence for the active bean."""
      bean = _get_active_bean(request, db)
      if not bean:
          return RedirectResponse(url="/beans", status_code=303)

      # Get all measurements for this bean, ordered by creation time
      measurements_raw = (
          db.query(Measurement)
          .filter(Measurement.bean_id == bean.id)
          .order_by(Measurement.created_at.asc())
          .all()
      )

      measurements = [
          {
              "taste": m.taste,
              "is_failed": m.is_failed,
              "created_at": m.created_at,
              "grind_setting": m.grind_setting,
          }
          for m in measurements_raw
      ]

      shot_count = len(measurements)
      convergence = _compute_convergence(measurements)

      # Determine optimizer phase
      optimizer = request.app.state.optimizer
      campaign = optimizer.get_or_create_campaign(bean.id, overrides=bean.parameter_overrides)
      from baybe.recommenders.pure.nonpredictive.sampling import RandomRecommender
      selected = campaign.recommender.select_recommender(
          batch_size=1,
          searchspace=campaign.searchspace,
          objective=campaign.objective,
          measurements=campaign.measurements,
      )
      optimizer_phase = "random" if isinstance(selected, RandomRecommender) else "bayesian"

      # Best measurement
      best_taste = None
      if measurements:
          non_failed = [m for m in measurements if not m["is_failed"]]
          if non_failed:
              best_taste = max(m["taste"] for m in non_failed)

      # Chart data (only if enough data)
      chart_data = None
      if shot_count >= 2:
          chart_data = _build_chart_data(measurements)

      return templates.TemplateResponse(
          request,
          "insights/index.html",
          {
              "active_bean": bean,
              "shot_count": shot_count,
              "convergence": convergence,
              "optimizer_phase": optimizer_phase,
              "best_taste": best_taste,
              "chart_data": chart_data,
          },
      )
  ```

  **Note:** Chart.js is loaded from CDN in the scripts block — only on the insights page (not globally). The `<script>` in `_progress_chart.html` runs after Chart.js is loaded because it's inline in the body and the CDN script is also in the body (after content, via {% block scripts %}).

  **Actually**, the `{% block scripts %}` renders *after* `{% block content %}`, so the Chart.js CDN script loads after the inline chart init script. This is a problem. Fix: Move the chart init into a `DOMContentLoaded` listener, or better: use `defer` attribute. But CDN scripts already execute in order.

  **Better approach:** Put the Chart.js CDN `<script>` tag at the TOP of `{% block scripts %}`, and the inline chart code is inside `_progress_chart.html` which is part of `{% block content %}`. Since both are in the body and execute in order, we need to ensure Chart.js loads before the inline script.

  **Simplest fix:** Move the Chart.js `<script src>` into the `<head>` section of the insights page, or wrap the chart init in `window.addEventListener('load', ...)`.

  **Recommended approach:** In `_progress_chart.html`, wrap chart init in `document.addEventListener('DOMContentLoaded', function() { ... })`. The CDN script in `{% block scripts %}` loads synchronously before the event fires. Actually this is still not guaranteed if the script is at the end of body.

  **Actual simplest approach:** Load Chart.js from CDN directly in `_progress_chart.html` before the chart init script, so loading order is guaranteed:
  ```html
  <script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.7/dist/chart.umd.min.js"></script>
  <script>
  // Chart init...
  </script>
  ```
  This is fine since it only loads on pages with charts. Remove the CDN script from `{% block scripts %}` and put it directly in the partial.
