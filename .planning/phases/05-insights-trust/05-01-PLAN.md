  <action>
  **New method on OptimizerService:**

  Add `get_recommendation_insights(self, bean_id, rec_dict, overrides=None) -> dict` method. This computes insight metadata for a recommendation:

  ```python
  def get_recommendation_insights(
      self,
      bean_id: str,
      rec_dict: dict,
      overrides: dict | None = None,
  ) -> dict:
      """Compute insight metadata for a recommendation.

      Returns dict with:
        - phase: "random" | "bayesian"
        - phase_label: str (human-readable)
        - explanation: str (contextual explanation)
        - predicted_mean: float | None
        - predicted_std: float | None
        - predicted_range: str | None (e.g. "4.5 - 8.5")
        - shot_count: int
      """
  ```

  Implementation:
  1. Get the campaign via `get_or_create_campaign(bean_id, overrides)`.
  2. Determine which recommender is active:
     ```python
     from baybe.recommenders.pure.nonpredictive.sampling import RandomRecommender
     meta_rec = campaign.recommender
     selected = meta_rec.select_recommender(
         batch_size=1,
         searchspace=campaign.searchspace,
         objective=campaign.objective,
         measurements=campaign.measurements,
     )
     is_random = isinstance(selected, RandomRecommender)
     ```
  3. Get measurement count from `campaign.measurements` (len if not empty, else 0).
  4. Compute phase and explanation:
     - If `is_random`: phase="random", phase_label="Random exploration", explanation="Exploring randomly — building initial understanding of the parameter space."
     - Else (Bayesian): phase="bayesian", phase_label="Bayesian optimization"
       - If shot_count < 5: explanation="Building a map of the flavor space — the model is starting to learn your preferences."
       - If shot_count >= 5: Check if the best taste in the last 3 measurements improved over the previous best. Use `campaign.measurements["taste"]` to check.
         - If improved recently: explanation="Zeroing in — recent shots are improving. The model is finding promising regions."
         - If not improved: explanation="Exploring new territory — looking for something better than the current best."
  5. Compute predicted taste (only if `not is_random` and `shot_count >= 2`):
     ```python
     import pandas as pd
     rec_df = pd.DataFrame([{k: rec_dict[k] for k in BAYBE_PARAM_COLUMNS}])
     with self._lock:
         stats = campaign.posterior_stats(rec_df)
     predicted_mean = round(float(stats["taste_mean"].iloc[0]), 1)
     predicted_std = round(float(stats["taste_std"].iloc[0]), 1)
     lo = round(max(1.0, predicted_mean - predicted_std), 1)
     hi = round(min(10.0, predicted_mean + predicted_std), 1)
     predicted_range = f"{lo} – {hi}"
     ```
     Wrap in try/except — if posterior_stats fails (insufficient data), set predicted_mean/std/range to None.
  6. Return the insights dict.

  **Important:** All BayBE calls must be within `self._lock` since campaign state is shared. The lock is already held during `recommend()`, so `get_recommendation_insights` should be called *after* `recommend()` completes (not inside it). The `_recommend` inner function in `recommend()` releases the lock after `campaign.recommend()`. So `get_recommendation_insights` acquires the lock separately.

  **Add import at top of file:**
  ```python
  from baybe.recommenders.pure.nonpredictive.sampling import RandomRecommender
  ```

  **Tests (tests/test_optimizer.py):**
  Add 3 tests:
  1. `test_insights_random_phase` — Create fresh campaign, get insights before any measurements → phase="random", predicted_mean=None
  2. `test_insights_bayesian_phase` — Add 2+ measurements, get insights → phase="bayesian", predicted_mean is not None
  3. `test_insights_with_improvement` — Add measurements where latest has highest taste → explanation contains "improving" or "Zeroing in"

  These tests can be synchronous — `get_recommendation_insights` is sync (no async needed, the BayBE calls are fast for posterior_stats).
